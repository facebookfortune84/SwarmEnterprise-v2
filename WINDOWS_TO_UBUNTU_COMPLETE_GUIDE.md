## 🎯 Three-Step Launch

### STEP 1: Windows 11 (5 Minutes)

**Run in PowerShell as Administrator:**

```powershell
# Enable Hyper-V
Enable-WindowsOptionalFeature -FeatureName Hyper-V -Online -All

# Restart when prompted
Restart-Computer
```

**Then create Ubuntu VM:**
- Use Hyper-V Manager GUI, OR
- Run the PowerShell script (see WINDOWS_UBUNTU_LAUNCH_GUIDE.md)

**Result:** Ubuntu 22.04 LTS VM boots to login screen

---

### STEP 2: Ubuntu VM Setup (5 Minutes)

**Install Ubuntu 22.04 LTS:**
1. Choose "Ubuntu Server (automated)"
2. Install with defaults
3. Enable SSH during install
4. Note VM's IP address
5. Reboot

**Result:** Ubuntu is running, SSH is enabled

---

### STEP 3: Launch SwarmEnterprise (15 Minutes)

**From Windows PowerShell:**

```powershell
ssh ubuntu@192.168.1.100  # Your VM IP
```

**Inside Ubuntu terminal:**

```bash
cd /home/ubuntu && \
wget https://raw.githubusercontent.com/rwv-techsolutions/swarmenterprise-v2/main/launch_ubuntu.sh && \
chmod +x launch_ubuntu.sh && \
bash launch_ubuntu.sh
```

**Wait 10-15 minutes. When you see "LAUNCH COMPLETE!" → Done! 🚀**

## 🛠️ Manual Commands Reference

### Windows PowerShell

```powershell
# Connect
ssh ubuntu@192.168.1.100

# Copy files
scp -r "file" ubuntu@192.168.1.100:/home/ubuntu/

# Manage VM
Start-VM -Name swarmenterprise-vm
```

### Ubuntu Bash

```bash
# Navigate to project
cd /opt/swarmenterprise

# Check services
docker compose ps
docker compose logs -f backend

# Restart backend
docker compose restart backend

# Stop/start all
docker compose down
docker compose up -d

# Test API
curl http://localhost:8000/health
```

## 🚨 Troubleshooting Quick Links

| Problem | Solution |
|---------|----------|
| Can't SSH to Ubuntu | Check VM IP in Hyper-V console |
| Docker not found | Run as `sudo` or add user to docker group |
| Port 8000 in use | Change `BACKEND_PORT` in `.env` |
| Database won't start | Check `docker compose logs postgres` |
| Backend won't start | Verify `.env` has `DATABASE_URL` |
| Health check timeout | Wait longer or check `docker logs` |

**Full troubleshooting:** See WINDOWS_UBUNTU_LAUNCH_GUIDE.md § Troubleshooting
